# STRUCTURE

## Root layout
- `package.json` contains frontend dependencies, scripts, and dev tooling.
- `tsconfig.json` configures TypeScript compilation.
- `vite.config.ts` configures Vite for React and TypeScript.
- `index.html` is the single-page app shell used by Vite and Tauri.
- `src/` contains the entire frontend application.
- `src-tauri/` contains the Tauri Rust backend and native app configuration.

## Frontend structure
- `src/main.ts`: React entrypoint and root render logic.
- `src/App.tsx`: main shell and tab/page router.
- `src/styles.css`: global CSS styles for the app UI.
- `src/config/ui.ts`: centralized theme tokens and `applyTheme()` helper.
- `src/components/`: reusable UI components.
  - `Header.tsx`
  - `Sidebar.tsx`
  - `UploadCard.tsx`
  - `OptionPanel.tsx`
- `src/pages/`: feature page modules.
  - `Home/index.tsx`
  - `History/index.tsx`
  - `Editor/index.tsx`
  - `SrtToAudio/index.tsx`
  - `Settings/index.tsx`

## Native wrapper structure
- `src-tauri/Cargo.toml`: Rust package manifest, dependencies, and library settings.
- `src-tauri/build.rs`: Tauri build script.
- `src-tauri/tauri.conf.json`: Tauri app configuration, window settings, bundle options.
- `src-tauri/src/lib.rs`: Tauri builder and command registration.
- `src-tauri/src/main.rs`: native app startup entrypoint.

## Assets and packaging
- `src/assets/`: frontend static assets directory.
- `src-tauri/icons/`: desktop app bundle icons.
- `dist/`: generated frontend build output (generated by Vite).

## Missing or placeholder structure
- No `tests/` folder or automated test files currently exist.
- No dedicated state or persistence layer is present.
- The feature pages are mostly scaffolds with placeholder content.
